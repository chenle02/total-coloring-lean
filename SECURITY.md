# Security policy

The initial library has no network service and does not parse external files.
Its relevant security surface is the development supply chain and any future
serialization or executable interface.

Do not open a public issue for a suspected vulnerability. Use
[GitHub private vulnerability reporting](https://github.com/chenle02/total-coloring-lean/security/advisories/new)
or email `chenle02@gmail.com` if that form is unavailable. Include a minimal
reproducer, the affected commit, and the expected impact. Security fixes target
the latest release.

A mistaken mathematical definition, missing hypothesis, or mismatch with the
companion certificate schema is scientifically important but is not normally a
security vulnerability. Such specification issues may be reported through a
public issue unless disclosure would create a concrete security risk.
